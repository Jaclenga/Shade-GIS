# Support

## What this repository supports today

Shade Study Builder is maintained as an actively evolving research software
prototype. The current support target is:

- local Streamlit use on a developer machine
- reproducible test runs with `pytest`
- builder workflows covered by the documented import, labeling, visualization,
  preview, and export paths

## Best way to ask for help

- Open a GitHub issue for reproducible bugs, broken docs, or feature requests.
- Include the operating system, Python version, input format, and exact steps to
  reproduce the issue.
- Attach small sample data when possible instead of screenshots alone.

## Response expectations

- Best-effort triage for new issues.
- Clarification requests may be needed before a bug can be reproduced.
- Early-stage or research-specific features may change quickly between commits.

## What is not guaranteed yet

- Formal release cadence
- Long-term API stability
- Backward compatibility for draft configuration formats
- Hosted production support outside locally reproducible workflows

## Security and sensitive data

Do not post credentials, private endpoints, or non-public study data in public
issues. If a report cannot be shared publicly, open a minimal public issue that
states a private follow-up is needed.
